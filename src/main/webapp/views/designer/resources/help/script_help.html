1. Auto Reload

Method : sdk.autoReload()

Arguments : Not required

This method should be written on Filters/ Grids/ Charts. Those connections will be reloaded automatically which has the filter key in their condition

2. Reload All Connections

Method : sdk.reloadAll()

Arguments : Not required

All connections will be reloaded

3. Reload from List

Method : sdk.reload( [ "C_1", "C_2" ] )

Arguments : An array of connection id's. Here C_1 and C_2 are the connection id's

Reloads those connections which are available in the array

4. Reload DataSet

Method : sdk.reloadDataset( [ "Bar_1" , "Column_2" ] );

Arguments : Components id's as array

Refresh the datasets of those components which are available in the array

NOTE: It will not reload the connection

26. Apply Filter

Method : sdk.applyDataSetFilter( "bar32", ["Year==2010","Quarter=='Q2'","Sales>=60"])

Arguments : Component id (e.g "bar32") ,Conditions written in form of array

Use the function to filter dataset of a component

examples : 

sdk.applyDataSetFilter("Bar1", ["Year==2010"]);
sdk.applyDataSetFilter("datagrid3", ["CompanyName=='Google' || CompanyName=='Amazon'" , "ProjectHrs>5000"]);
sdk.applyDataSetFilter("Bar2", ["Year>=2010", "Month=='February'", "Week=='2'"]);
sdk.applyDataSetFilter("Bar3", ["Year=={radioYr.Value}", "Week>=2", "Week!=4"]);
sdk.applyDataSetFilter("Grid2", ["ProjectName=='{filter2.ProjectName}'", "EmployeeCount>={hslider2.Value}"]);
If field's name is having space (eg: "Overall Expense" , "Total Profit"):
sdk.applyDataSetFilter("Bar1", ["order by [Total Profit] asc"]);             ( if [Total Profit] is category field )
sdk.applyDataSetFilter("Bar1", ["order by [Total Profit] numasc"]);     ( if [Total Profit] is series field )
For string sort (use for Category):
sdk.applyDataSetFilter("Bar1", ["order by [City] asc"]);
sdk.applyDataSetFilter("Bar1", ["order by [City] desc"]);
For numeric data sort (use for Series):
sdk.applyDataSetFilter("Bar1", ["order by [Total Profit] numasc"]);
sdk.applyDataSetFilter("Bar1", ["order by [Total Profit] numdesc"]);
To Sort the dataset and limit number of records, write condition like this:
sdk.applyDataSetFilter("Bar1", ["order by Sales asc", "limit 10"]);
All the conditions written in array, are executed with AND(&&) Operator, If you need to use OR(||) operator: Please write the filter conditions like this: 
sdk.applyDataSetFilter("datagrid3", ["CompanyName=='Google' || CompanyName=='Yahoo'" , "ProjectHrs>5000"]);

Available Operators are: > , < , == , != , >= , <= , || , &&

Every Conditions should be enclosed in DOUBLE QUOTES: "CompanyName=='Google'" is a valid condition

Field name shouldn't be kept in SINGLE QUOTE: "'CompanyName'=='Google'" is invalid condition

Field Name should be valid javascript variable names

Some invalid column names are:
  • Javascript identifiers ( var, boolean, number, function, prototype, new, try, for, Object, this etc.)
Preference of operations on dataset :
  1. Calculated fields
  2. Append additional rows
  3. Automanipulation
  4. Filter
  5. Aggregation
Limitations:

  • If a chart has AutoManipulator enabled, filter is applicable only on GroupBy field and Dynamically-Generated-Fields

    Valid case: 
    sdk.applyAutoManipulator("column14", "Year", "Month", "Expense", "#A0D468,#9087EE,#F89406,#EF4836,#6C7A89,#4ECDC4,#F5D76E,#BF55EC");
sdk.applyDataSetFilter( "column14", ["Year>2011", "Jan>100"] );
    Invalid Case: 
    sdk.applyAutoManipulator("column14" , "Year", "Month", "Expense", "#A0D468,#9087EE,#F89406,#EF4836,#6C7A89,#4ECDC4,#F5D76E,#BF55EC");
sdk.applyDataSetFilter( "column14", ["Year>2011", "Month=='Jan'"] );
    "Month=='Jan'" is invalid condition here as the field "Month" will not be available in the resultant dataset after automanipulation operation

  • If a chart has AutoManipulator disabled, filter is applicable on GroupBy fields and Original-Dataset-Fields which are coming from connection

    Valid case: 
    sdk.applyAutoManipulator("column14", "Year", "Month", "Expense", "#A0D468,#9087EE,#F89406,#EF4836,#6C7A89,#4ECDC4,#F5D76E,#BF55EC");
sdk.applyDataSetFilter( "column14", ["Year>2011", "Month=='Jan'"] );
    Invalid Case: 
    sdk.applyAutoManipulator("column14", "Year", "Month", "Expense", "#A0D468,#9087EE,#F89406,#EF4836,#6C7A89,#4ECDC4,#F5D76E,#BF55EC");
sdk.applyDataSetFilter( "column14", ["Year>2011", "Jan>100"] );
    "Jan>100" is invalid condition here as the field "Jan" is not coming from the connection

28. Apply Aggregation

Method : sdk.applyAggregation("bar32", "Year", "Revenue,Expenses", "SUM,SUM")

Arguments - Component id (e.g "bar32") , Group by node (e.g "Year") , Fields to be aggregated (e.g. "Sales,Profit") , Operations to be applied (e.g. "COUNT,SUM")

Use the function to apply aggregation on dataset of chart/ grid. These are he available aggregation operations - SUM, AVG, MAX, MIN, COUNT

Group By node is optional and configured as following:

  • if left blank, only single record will come in result with aggregated values for mentioned Fields

    sdk.applyAggregation("bar32", "", "Revenue", "SUM");

  • If one group by field is given, result will be grouped according to unique values of the fields(Year in the example) 

    sdk.applyAggregation("bar32", "Year", "Revenue", "SUM");

  • If more then one field is given, result will be grouped by last field(Month field in the example) and then grouped by previous field(Year field in example) 

    sdk.applyAggregation("bar32", "Year,Month", "Revenue", "SUM");

38. apply Language Mapping

Method : sdk.applyLanguageMapping();

Arguments - Configuration Object

  • isEnabled - to enable language setting in that dashboard
  • language - language for a user, this can be set as 'all' or can be taken from user-custom-properties
  • name - name of the Language Setting configuration which is available in the Admin Panel
  • hideBrackets - set true, if {curly brackets} are not required when there are no mapping available for some keys

Details:

  • This method should be written at the dashboard script, Language mapping service will return the mapping json when dashboard loads.
  • Enclose the mapping keys in curly brackets {} wherever required in a component.

Applicable at following places:

  • Chart - Title, Subtitle, Axis descriptions, Legends, Tooltips
  • Datagrid - headers
  • Bullet/ Filter/ Gauge/ Semi Gauge and Url Button - title
  • Label: Dynamic property should be true
sdk.applyLanguageMapping( {"isEnabled": "true", "language": "hi-IN", "name": "Sales2018Dashboard", hideBrackets: "false"} );

SDK methods : Usage and examples

Script can be written at these places
  1. On Component: On click of a data point of charts , click on a row of datagrid, change of filters, click on label/ icon/ rectangle etc.
  2. On Connection load: Script will be executed just after the connection fetches the data
Available Javascript Objects in scripting

  1. sdk

    It is a desiger SDK Object which has many inbuilt functions used to achieve various functionalities in dashboard.
    Syntax and usage of those methods are explained above

  2. changedItem

    Javascript Object which contains attributes.
    i.e if script is written for a LineChart, When we click on a data point, script will be executed and changedItem will be referred to lineChart
    "changedItem.key": id of the lineChart
    "changedItem.attributes": associated values to that data point

Script to use "context" variable

  1. Context global variable is used to fetch the properties loaded from portal such as user name,email, custom property etc. Below listed are the available attributes for an user.
    • dashboard_emailid: "username@domain.com"
    • dashboard_uniqueid: username@domain.com
    • dashboard_id: "user id"
    • dashboard_fullname: "Last Name First Name"
    • dashboard_username: "user name"
    • dashboard_user: Object which has all the user properties
    • Examples: 
      var cntx = sdk.getGlobalVariable( "context" );
var un = cntx.attributes.dashboard_username;
sdk.setValue("label1", "Welcome " + un);
var at = cntx.attributes.dashboard_authtype;
sdk.setValue("label2", "authType " + at);
      var cntx = sdk.getGlobalVariable( "context" );
var user = cntx.attributes.dashboard_user;
sdk.setValue("label3", "Welcome " + user.userName);
sdk.setValue("label4", "authType " + user.authType);
  2. If the user is having a custom field in the portal, that also can be fetched in dashboard in the following format
    • Examples: by method getUserCustomProperty 
      var cpd = sdk.getUserCustomProperty("Department");
sdk.setValue("label1", "Dept: " + cpd);
var cpz = sdk.getUserCustomProperty("ZONE");
sdk.setValue("label2", "Zone: " + cpz);
var cpe = sdk.getUserCustomProperty("Emergency Contact Number");
sdk.setValue("label3", "Emergency Contact Number: " + cpe);
    • Examples: by key user_CustomPropertyName 
      var cntx = sdk.getGlobalVariable( "context" );
var cpd = cntx.attributes.user_Department;
sdk.setValue("label1", "Dept: " + cpd);
var cpz = cntx.attributes.user_ZONE;
sdk.setValue("label2", "Zone: " + cpz);
var cpe = cntx.attributes["user_Emergency Contact Number"];
sdk.setValue("label2", "Emergency Contact Number: " + cpe);
  3. If the dashboard has Dashboard parameters set from the portal, that also can be fetched in dashboard in the following format 
    var cntx = sdk.getContext( "dashboard_dashboardparameters" );
var dpl = cntx.lang;
console.log(dpl);

Filter Saver Cascading

Write below script on the dashboard 

sdk.getWidget( 'filtersaver1' ).m_filtersavermap = {
	"popupHeaderText": "Filter Saver",	// to edit the header text of modal pop-up
	"popupAddCriteriaText": "Add Criteria",	// to edit the Add criteria panel text
	"popupListCriteriaText": "Criteria List",	// to edit the list criteria panel text
	"enableCheckBox": "false",	// enable-disable checkbox, false by default
	"listFilterComponents": ["filter1","filter2"],	// to list only those filters whose value has to be saved.
	"cascadingDefinition": ["filter2","filter1","filter3"]	// arrange filters in the cascading order.
}

"listFilterComponents" can be used to list all those filters whose value has to be saved either when checkbox is enabled,disabled or cascading scenario.

Mandatory condition for cascaded filter:
* Don't use sdk.updateGlobalVariable() script, enable global variable from property palette.
* "cascadingarraydef" is required when filter are cascaded. In the above definition, index of all the filters(filter3 and filter1) will be updated
in descending order without redrawing it, redraw will be triggered at last only for the 1st filter mentioned in the definition (ie, filter2) 
that will result in the redrawing of all cascaded filters. In result, it will avoid the multiple service loading.
